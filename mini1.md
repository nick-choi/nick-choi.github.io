# 전투 시련 기획 요구사항 (개발용 요약)

## 1. 전투 시련 진입

### 발생 조건

- 메인 로비에서 **전투 시련 발생**
- 플레이어가 시련 진입 시 전투 화면으로 이동

### 진입 연출

- 전투 화면 전환 연출 재생
- 연출 종료 후 전투 UI 노출


## 2. 전투 화면 구성

### 화면 구조

- **1인칭 시점 전투**
- 정면에 적 캐릭터 표시
- 플레이어 캐릭터는 직접 보이지 않음
- 배경은 단순 처리 가능 (검은 배경 또는 최소 연출)

### UI 요소

### 적 정보

- 적 HP Bar
- 적 캐릭터 이미지

### 플레이어 정보

- 플레이어 HP Bar
- 플레이어 허기(스태미나) Bar

### 선택지(Action)

각 선택지는 아래 정보 포함:

- 성공 확률(%)
- 예상 데미지 범위
- 선택 가능 여부

예시:

- 90% / 데미지 1~2
- 50% / 데미지 5~6
- 15% / 데미지 10~15

---

## 3. 턴 진행 규칙

### 기본 흐름

`선택지 선택 → 유저 공격 → 적 반격 → 상태 반영 → 다음 선택`

### 상세 플로우

1. 플레이어가 선택지 1개 선택
2. 선택 성공 판정
3. 유저 공격 연출 실행
4. 적 반격 실행
5. 변경된 수치 적용
6. 다시 선택 단계 진입

---

## 4. 공격 성공 처리

### 성공 판정 시

- 선택지 확률 기반 성공
- 선택지에 정의된 데미지 적용

### 연출

### 유저 공격

- 적 타격 이펙트
- 피해 숫자 표시
- 허기 감소 숫자 표시

### 적 반격

- 적 확대/돌진 등 피격 표현
- 화면 테두리 붉은 플래시
- 플레이어 피해 숫자 표시

### 수치 반영

- 적 HP 감소
- 플레이어 HP 감소
- 플레이어 허기 감소
- UI 즉시 갱신

---

## 5. 공격 실패 처리

### 실패 판정 시

- 유저 공격 MISS 처리
- 적 피해 없음

### 연출

- `MISS` 텍스트 표시
- 공격 실패 이펙트

### 이후 흐름

- 적 반격 진행
- 플레이어 HP/허기 감소 적용
- 상태 반영 후 선택 단계 복귀

---

## 6. 전투 종료 조건

### 시련 성공

조건:

- 적 HP <= 0

처리:

1. 적 쓰러짐 연출 재생
2. 전투 종료
3. 메인 로비 복귀
4. 시련 성공 팝업 출력

팝업 내용:

- 적 처치 이미지
- 성공 문구
- 닫기 버튼

### 시련 실패

조건:
- 플레이어 HP <= 0 ?

처리:
- 실패 팝업 또는 별도 결과 화면 필요
