# Onscreen Debug Messages
## 효율적인 개발을 위한 환경 세팅 (작은 맵 활용)

* **오픈 월드의 한계**: 맵이 너무 크면 액터가 플레이어와 멀리 떨어졌을 때 중요도(Relevance) 문제로 코드가 실행되지 않아 로그 확인이 어렵습니다.
    
* **해결법**: 학습 및 개발 단계에서는 시작용 콘텐츠(Starter Content) 폴더에 있는 **`Minimal_Default`** 같은 작은 맵을 사용하는 것이 좋습니다.
    
* **주의점**: 액터의 C++ 코드가 실행되려면 **해당 액터(`BP_Item`)가 반드시 월드(레벨)에 배치되어 있어야 합니다.**

## 블루프린트의 Print String 심화: `Key`와 `Tick`
블루프린트의 `Print String` 노드는 하단 상세 옵션을 펼치면 `Key` (Name 타입)라는 숨겨진 입력 파라미터가 있습니다.

### ① `Key`의 역할 (메시지 중복 방지 및 교체)

* **기본 상태 (`Key = None`)**: 새로운 메시지가 위로 계속 쌓이며 스크롤됩니다 (화면 스패밍 현상 발생).
    
* **Key 지정 시 (`Key = 1`)**: 동일한 Key 값을 가진 다른 `Print String`이 실행되면, 기존 메시지가 화면에서 사라지고 새로운 메시지가 그 자리를 대체(Overwrite)합니다.

### ② `Tick` 이벤트와 `Delta Seconds` 출력

* **`Tick` 이벤트**: 매 프레임(화면이 갱신될 때마다 보통 초당 60~120회)마다 호출됩니다.

* **`Delta Seconds`**: 이전 프레임에서 현재 프레임까지 걸린 시간(밀리초 단위의 매우 작은 실수값)입니다. 프레임률에 따라 계속 가변합니다.
    
* **작동 방식**: `Delta Seconds`를 `Print String`의 `In String`에 연결하면 중간에 'Double(또는 Float)을 String으로 변환하는 노드'가 자동으로 생성됩니다. 이때 `Key` 값을 고정해 두면 화면이 메시지로 도배되지 않고 실시간으로 변하는 수치만 깔끔하게 관찰할 수 있습니다.

## C++에서 화면에 디버그 메시지 띄우기: `GEngine`
C++에서는 `UE_LOG`를 통해 출력 로그 창에만 글자를 띄울 수도 있지만, 블루프린트의 `Print String`처럼 게임 화면에 바로 글자를 띄우려면 **`GEngine`** 전역 포인터를 사용해야 합니다.

### ① 널 포인터 검사 (Null Pointer Check)

`GEngine`은 전역 포인터 변수이므로 시스템이 초기화되기 전이나 특정 상황에 메모리 주소가 `0(Null)`일 수 있습니다. 안전한 프로그래밍을 위해 **반드시 `if (GEngine)` 문으로 유효성을 먼저 검사**해야 에디터 크래시(다운 현상)를 방지할 수 있습니다.

### ② `AddOnScreenDebugMessage` 함수 사용법

```cpp
if (GEngine)
{
  GEngine->AddOnScreenDebugMessage(
    1,                 
    // 1. Key (정수형 변수, 동일 Key는 메시지 교체)
    60.f,              
    // 2. 화면에 표시될 시간 (초 단위 Float, 암시적 형변환 방지를 위해 .f 사용)
    FColor::Cyan,      
    // 3. 텍스트 색상 (FColor 클래스의 정적 변수 활용)
    FString("Item on screen message!") 
    // 4. 출력할 문자열 (FString 생성자 활용)
  );
}
```
* **오버로드(Overload)**: 이 함수는 2가지 버전이 존재하며, 입력하는 인자값에 따라 알맞은 버전이 실행됩니다.
    
* **기본값 파라미터(Default Arguments)**: 위 코드의 4가지 필수 인자 뒤에 오는 파라미터(예: `bNewerOnTop` 등)들은 엔진 내부적으로 기본값이 지정되어 있으므로 생략해도 무방합니다.

# Formatting Strings
## 기초 환경 설정 (시작 맵 변경)
오픈 월드 대신 작은 맵(`Minimal_Default`)에서 매번 편하게 테스트할 수 있도록 에디터 설정을 변경합니다.

* **방법**: `Edit` -> `Project Settings` -> `Maps & Modes`에서 **Editor Startup Map**을 `Minimal_Default`로 변경합니다. 다음부터 에디터를 껐다 켜도 자동으로 이 작은 맵이 열립니다.

## C++ `UE_LOG`에서 실수(`float`) 출력하기
C++의 `UE_LOG` 매크로는 세 번째 파라미터부터 가변 인자(Variable Arguments)를 받습니다. 즉, 문자열 뒤에 컴마(`,`)를 찍고 원하는 변수를 여러 개 넘길 수 있습니다.

### ① 형식 지정자 (Format Specifier) 사용
* 변수를 문자열 내부에 넣으려면 C/C++ 언어에서 유래한 **형식 지정자**를 사용해야 합니다.
* **`%f`**: 소수점 아래 실수를 표현하는 지정자입니다. (대문자 `%F`도 가능)

### ② 구현 코드 예시 (`Tick` 함수 내)
```cpp
void AItem::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // %f 자리에 뒤에 있는 DeltaTime 값이 대입되어 출력 로그에 찍힙니다.
    UE_LOG(LogTemp, Warning, TEXT("Delta time: %f"), DeltaTime);
}
```
* **결과**: 출력 로그 창에 매 프레임 변하는 `DeltaTime` 수치가 실시간으로 도배(Spamming)되며 출력됩니다.

### 화면 디버그 메시지에서 포맷팅하기: `FString::Printf`
화면에 글자를 띄우는 `GEngine->AddOnScreenDebugMessage()`는 문자열 자체를 포맷팅하는 기능이 내장되어 있지 않습니다. 따라서 **미리 포맷팅된 `FString` 오브젝트를 만들어서** 전달해야 합니다.

### ① `FString::Printf` 함수 사용
언리얼 엔진은 C++ 표준 라이브러리의 `std::string` 대신 자체적인 **`FString`** 타입을 사용합니다. 이 클래스가 제공하는 정적(Static) 함수인 `Printf`를 사용하면 문자열을 쉽게 포맷팅할 수 있습니다.

### ② 구현 코드 예시
```cpp
if (GEngine)
{
    // 1. Printf를 사용해 포맷팅된 FString 생성 후 message 변수에 저장
    FString message = FString::Printf(TEXT("Delta Time: %f"), DeltaTime);

    // 2. 생성된 message를 화면에 출력 (Key를 1로 지정해 메시지가 겹쳐서 갱신되도록 설정)
    GEngine->AddOnScreenDebugMessage(1, 60.f, FColor::Cyan, message);
}
```

## 문자열(`FString`)을 형식 지정자로 출력할 때의 필수 규칙: 포인터(`*`) 연산자
디버깅을 할 때 숫자가 아니라, 액터의 이름 같은 **또 다른 문자열을 포맷팅 안에 넣고 싶을 때**는 아주 중요한 규칙이 있습니다.

### ① 형식 지정자 `%s`와 역참조(`*`) 연산자

* 문자열을 위한 형식 지정자는 `%s`를 사용합니다.
* 액터의 이름을 가져오는 `GetName()` 함수는 `FString` 타입을 반환합니다.
* **핵심 규칙**: `%s` 자리에 `FString` 변수를 그대로 넣으면 컴파일 에러가 나거나 오작동합니다. 반드시 변수 이름 앞에 역참조 연산자(`*`)를 붙여야 합니다.

```cpp
FString name = GetName(); // 액터의 내부 고유 이름을 가져옴

// ⚠️ 중요: name 앞에 반드시 '*'를 붙여야 합니다!
FString message = FString::Printf(TEXT("Item Name: %s"), *name);
```

### ② 왜 `*`를 붙여야 할까? (내부 메커니즘)

언리얼 엔진의 `FString` 클래스는 내부적으로 `*` 연산자가 오버로딩(재정의)되어 있습니다. `FString` 앞에 `*`를 붙이면, 언리얼의 유니코드 문자 배열 포인터 타입인 `TCHAR*` (C 스타일 문자열)을 반환해 줍니다. `Printf`나 `UE_LOG` 구조상 옛날 방식의 C 스타일 문자열 주소값을 요구하기 때문에 이 변환 과정이 필수적입니다.

## 언리얼 엔진의 액터 고유 이름 확인
위 코드를 적용하고 월드에 `BP_Item`을 **Alt + 드래그**로 복사하여 2개를 배치한 뒤 실행하면 다음과 같은 결과를 볼 수 있습니다.

* **출력 결과 예시**: `BP_Item_C_0`, `BP_Item_C_1`
    
* **이유**: 언리얼 엔진은 월드에 존재하는 모든 오브젝트를 고유하게 식별해야 하므로, 동일한 블루프린트여도 내부적으로 뒤에 고유한 숫자나 문자를 자동으로 덧붙여(Append) 관리합니다.
