# Drawing Debug Spheres
## 디버그 모양(Debug Shapes)이 필요한 이유
* 메시(Mesh)가 없는 액터(예: 아이템, 스폰 포인터, 트리거 등)는 게임을 실행했을 때 눈에 보이지 않습니다.
* 개발 과정에서 이 invisible(보이지 않는) 액터들이 **"월드 어디에, 어떤 크기로 존재하는지"** 시각적으로 확인하기 위해 디버그 모양을 사용합니다.
* *주의: 디버그 모양은 개발용 도구이므로, 최종 패키징 게임에서는 성능을 위해 사용하지 않습니다.*

## 블루프린트에서 디버그 구체 그리기
`Begin Play` 이벤트에 **`Draw Debug Sphere`** 노드를 연결하여 구현합니다.

* **Center (중심점)**
    * 구체가 그려질 3D 좌표입니다. (X, Y, Z 벡터 타입)
    * `Get Actor Location` 노드를 사용해 해당 액터의 위치를 중심점으로 지정할 수 있습니다.
        
* **Radius (반지름)**
    * 구체의 크기입니다. (Float 타입)
    
* **Segments (세그먼트)**
    * 구체를 구성할 선의 개수입니다. 값이 클수록 구체가 부드럽고 둥글게 보이지만 성능을 더 소모합니다. (기본값 12, 24 등으로 변경 가능)
    
* **Line Color (선 색상)**
    * `Linear Color` 타입으로, 컬러 픽커를 통해 원하는 색상을 지정합니다.
    
* **Duration (지속 시간)**
    * 구체가 화면에 몇 초 동안 머무를지 설정합니다.
    
* **Thickness (두께)**
    * 디버그 선의 두께를 조절합니다.

## C++에서 디버그 구체 그리기
### ① 헤더 파일 포함 (Include)
C++에서 디버그 함수를 사용하려면 소스 파일(`.cpp`) 상단에 관련 헤더를 포함해야 합니다.
```cpp
#include "DrawDebugHelpers.h"
```

### ② 안전한 월드 포인터 체크 및 구현
`Begin Play` 함수 내부에서 다음과 같이 작성합니다. `GetWorld()`가 유효한지(Null이 아닌지) `if`문으로 먼저 체크하는 것이 좋은 C++ 습관입니다.

```cpp
// 1. 액터의 위치 가져오기 (FVector 타입)
FVector Location = GetActorLocation();

// 2. 월드 포인터 가져오기 및 검증
UWorld* World = GetWorld();
if (World)
{
    // 3. 디버그 구체 그리기 함수 호출
    DrawDebugSphere(
        World,       // 현재 월드 포인터
        Location,    // 중심점 위치 (FVector)
        25.0f,       // 반지름 (Radius)
        24,          // 세그먼트 수 (int32 - 언리얼에서 보장하는 32비트 정수형)
        FColor::Red, // 색상 (FColor)
        false,       // 영구 지속 여부 (bPersistentLines)
        30.0f        // 지속 시간 (Lifetime, 영구 지속이 false일 때 적용)
    );
}
```
* **참고 (`int32`):** C++ 기본 `int`는 플랫폼마다 크기가 달라질 수 있어, 언리얼 엔진에서는 항상 32비트를 보장하는 자체 타입인 `int32`를 사용합니다.

## 매크로(Macro)를 이용한 C++ 코드 효율화
`DrawDebugSphere` 함수는 매번 입력해야 하는 매개변수(인자)가 너무 많습니다. 이를 간소화하기 위해 **함수형 매크로**를 정의해 사용할 수 있습니다.

### ① 매크로 정의 (`#define`)
반지름, 세그먼트, 색상 등을 고정값으로 두고, 오직 '위치(Location)'만 인자로 받는 `DRAW_SPHERE` 매크로를 만듭니다. (가장 범용적인 헤더 파일에 정의하면 편리합니다.)
```cpp
// 예: 프로젝트의 메인 헤더 파일(예: Slash.h)에 작성
#define DRAW_SPHERE(Location) if (GetWorld()) DrawDebugSphere(GetWorld(), Location, 25.f, 12, FColor::Red, true);
```

* `bPersistentLines`를 `true`로 설정했기 때문에, 이 매크로로 그린 구체는 사라지지 않고 게임 내내 유지됩니다.

### ② 매크로 사용하기
소스 파일(`.cpp`)에서 해당 헤더를 인클루드하면, 단 한 줄로 깔끔하게 디버그 구체를 그릴 수 있습니다.

```cpp
#include "Slash/Slash.h" // 매크로가 정의된 헤더 포함

void AItem::BeginPlay()
{
    Super::BeginPlay();

    FVector Location = GetActorLocation();
    DRAW_SPHERE(Location); // 복잡한 인자 없이 간단하게 호출 가능!
}
```

이 방식을 사용하면 코드가 훨씬 간결해지며, 여러 다른 클래스에서도 액터의 위치를 간편하게 시각화하여 디버깅할 수 있습니다.

# Drawing Debug Lines
## 디버그 선(Debug Line)이 필요한 이유와 벡터 수학

* **필요성:** 디버그 구체(Sphere)가 액터의 '위치'를 보여준다면, 디버그 선(Line)은 액터가 **"어느 방향을 바라보고 있는지(방향성)"** 시각적으로 나타낼 때 매우 유용합니다.
    
* **시작점과 끝점:** 선을 그리려면 시작점(Start)과 끝점(End) 좌표가 필요합니다.
    * **시작점:** 액터의 현재 위치(`Get Actor Location`)
    * **끝점 계산 (벡터 수학):** 액터의 현재 위치에 '앞방향'을 뜻하는 포워드 벡터(Forward Vector)를 더해 구합니다.
        
### 중요한 벡터 개념: 벡터의 스케일링(Scaling)
1.  `Get Actor Forward Vector`가 반환하는 벡터는 길이가 `1`인 단위 벡터(Normalized Vector)입니다.
2.  언리얼 엔진의 기본 단위는 센티미터(cm)이므로, 위치에 포워드 벡터를 그대로 더하면 겨우 **1cm 길이의 아주 짧은 선**이 그려져 눈에 보이지 않습니다.
3.  따라서 포워드 벡터에 **곱셈(소수점/정수)을 백 배 단위(예: `* 100`)로 해주어 길이를 늘린(스케일링) 후**, 액터 위치에 더해주어야 눈에 보이는 충분한 길이(1m)의 선을 얻을 수 있습니다.

## 블루프린트에서 디버그 선 그리기
`Begin Play` 이벤트에 **`Draw Debug Line`** 노드를 연결하여 구현합니다.

* **Line Start**
    * `Get Actor Location` 연결
    
* **Line End**
    * `Get Actor Forward Vector` 노드 드래그 ➡️ `Multiply(*)` 노드로 100을 곱함 ➡️ `Add(+)` 노드로 액터 위치와 더한 후 연결
    
* **Line Color**
    * 원하는 색상 지정 (예: Red)
    
* **Duration**
    * 60초 지정

* **Thickness**
    * 선의 두께 지정

## C++에서 디버그 선 그리기
### ① 변수 정리 및 함수 호출
이전 강의에서 포함한 `#include "DrawDebugHelpers.h"`를 그대로 사용하며, `Begin Play` 내부에서 다음과 같이 작성합니다.
```cpp
UWorld* World = GetWorld();
if (World)
{
    // 1. 위치와 포워드 벡터 가져오기
    FVector Location = GetActorLocation();
    FVector Forward = GetActorForwardVector();

    // 2. C++ 디버그 선 그리기 함수 호출
    DrawDebugLine(
        World,
        Location,    // 시작점 (Start)
        Location + (Forward * 100.f),  // 끝점 (End): 포워드 벡터를 100배 스케일링하여 더함
        FColor::Red, // 색상
        true,        // 영구 지속 여부 (bPersistentLines)
        -1.f,        // 영구 지속이 true이므로 수명은 의미 없음 (-1.0f)
        0,           // 깊이 우선순위 (Depth Priority: 0은 최상단에 그려짐)
        1.f          // 선 두께 (Thickness)
    );
}
```
* **C++의 장점:** 블루프린트처럼 복잡하게 노드를 잇지 않고, 연산자 오버로딩 덕분에 `Location + (Forward * 100.f)`와 같이 수학 공식을 쓰듯 직관적이고 깔끔하게 작성할 수 있습니다.

## 매크로(Macro)를 이용한 C++ 코드 효율화
구체 매크로와 마찬가지로 인자가 많은 `DrawDebugLine`을 매크로로 만들어 단 한 줄로 축약합니다.

### ① 매크로 정의 (`#define`)
공용 헤더 파일(예: `Slash.h`)에 시작점과 끝점만 인자로 받는 `DRAW_LINE` 매크로를 정의합니다.
```cpp
// Slash.h 파일에 추가
#define DRAW_LINE(StartLocation, EndLocation) if (GetWorld()) DrawDebugLine(GetWorld(), StartLocation, EndLocation, FColor::Red, true, -1.f, 0, 1.f);
```

### ② 매크로 사용하기
아이템 소스 파일(`.cpp`)에서 지저분한 코드를 모두 지우고 매크로를 호출하여 깔끔하게 정리합니다.
```cpp
#include "Slash/Slash.h" // 매크로 헤더 포함

void AItem::BeginPlay()
{
    Super::BeginPlay();

    FVector Location = GetActorLocation();
    FVector Forward = GetActorForwardVector();

    // 매크로를 사용해 구체와 선을 동시에 쉽게 시각화
    DRAW_SPHERE(Location);
    DRAW_LINE(Location, Location + Forward * 100.f); 
}
```

이로써 액터가 보이지 않더라도 월드 상에서 **어디에 위치(Sphere)해 있고, 어느 방향(Line)을 바라보는지**를 C++ 환경에서 아주 간결한 코드로 완벽하게 디버깅할 수 있게 되었습니다.

# Drawing Debug Points
## 디버그 포인트(Debug Point)의 특징
* 디버그 선만으로는 벡터의 시작점(꼬리)과 끝점(머리)을 구별하기 어렵기 때문에, 끝점 좌표에 **디버그 포인트**를 찍어 화살표 같은 효과를 냅니다.
    
* **특징 (화면 크기 고정):** 디버그 포인트는 화면에서의 절대적인 픽셀 크기가 고정되는 특성이 있습니다. 따라서 카메라를 멀리 떨어뜨리면(Zoom out) 상대적으로 구체는 작아지지만 포인트는 그대로여서 커 보이고, 가까이 다가가면(Zoom in) 포인트가 너무 작아져서 거의 보이지 않게 됩니다.

## 블루프린트에서 디버그 포인트 그리기
`Begin Play` 이벤트에 **`Draw Debug Point`** 노드를 연결하여 구현합니다.

* **Position (위치)**
    * 점이 찍힐 3D 좌표입니다.

* **Size (크기)**
    * 점의 크기를 지정합니다. (영상에서는 15 사용)

* **Color / Duration**
    * 각각 색상과 지속 시간을 지정합니다.

## C++에서 디버그 포인트 그리기 및 매크로화
### ① 기본 구현 (C++)
`Begin Play` 내부에서 월드가 유효한지 체크한 후, 벡터의 끝점 위치(`Location + Forward * 100.f`)에 점을 그립니다.
```cpp
UWorld* World = GetWorld();
if (World)
{
    FVector Location = GetActorLocation();
    FVector Forward = GetActorForwardVector();

    // 디버그 포인트 그리기 (크기: 15.f, 영구 지속: true)
    DrawDebugPoint(World, Location + (Forward * 100.f), 15.f, FColor::Red, true);
}
```
* **팁:** C++에서 매크로(예: `DRAW_SPHERE`) 뒤에 세미콜론 `;`을 붙이지 않고 줄바꿈을 하면, 비주얼 스튜디오가 코드가 끝나지 않은 것으로 오해해 자동 들여쓰기(Indentation) 오류가 날 수 있습니다. 매크로 정의 내부에 세미콜론이 있더라도 코드의 가독성을 위해 호출할 때 뒤에 `;`을 붙여주는 것이 좋습니다.

### ② 디버그 포인트 매크로 정의 (`#define`)
공용 헤더 파일(`Slash.h`)에 위치값만 인자로 받는 `DRAW_POINT` 매크로를 정의합니다.
```cpp
// Slash.h에 추가
#define DRAW_POINT(Location) if (GetWorld()) DrawDebugPoint(GetWorld(), Location, 15.f, FColor::Red, true);
```

## 응용: 다중 행 매크로(Multi-line Macro)로 `DRAW_VECTOR` 만들기
선(`DRAW_LINE`)과 점(`DRAW_POINT`)을 따로 호출하는 대신, 시작점과 끝점만 넣어주면 알아서 선을 긋고 끝에 점을 찍어주는 **통합 벡터 디버그 매크로**를 만듭니다.

### ① 역슬래시(`\`)를 이용한 다중 행 매크로 정의
C++ 매크로는 기본적으로 한 줄로만 작성해야 하지만, 줄 끝에 역슬래시(`\`)를 붙여주면 컴파일러가 다음 줄도 같은 매크로 내용으로 인식합니다. 중간에 중괄호`{}`를 사용해 여러 줄의 코드를 묶어줍니다.
```cpp
// Slash.h 파일에 작성하는 다중 행 매크로
#define DRAW_VECTOR(StartLocation, EndLocation) \
if (GetWorld()) \
{ \
    DrawDebugLine(GetWorld(), StartLocation, EndLocation, FColor::Red, true, -1.f, 0, 1.f); \
    DrawDebugPoint(GetWorld(), EndLocation, 15.f, FColor::Red, true); \
}
```
* **주의:** 매크로의 맨 마지막 줄(여기서는 닫는 중괄호 `}`) 끝에는 역슬래시(`\`)를 붙이지 않습니다.

### ② 최종 코드 적용하기
이제 `.cpp` 소스 코드가 믿을 수 없을 정도로 깔끔해집니다.
```cpp
#include "Slash/Slash.h" // 매크로 헤더 포함

void AItem::BeginPlay()
{
    Super::BeginPlay();

    FVector Location = GetActorLocation();
    FVector Forward = GetActorForwardVector();

    // 단 두 줄로 구체, 선, 벡터 끝점까지 모두 시각화 완료!
    DRAW_SPHERE(Location);
    DRAW_VECTOR(Location, Location + Forward * 100.f);
}
```

# Custom Header Files
## 1. 헤더 파일 분리(모듈화)가 필요한 이유
*   이전 강의에서는 `Slash.h`라는 프로젝트 전역 헤더 파일에 디버그 매크로(`DRAW_SPHERE`, `DRAW_VECTOR` 등)를 정의했습니다.
    
*   하지만 이런 방식은 다른 클래스에서 `Slash.h`를 인클루드할 때 필요 없는 디버그 기능까지 전부 포함하게 만들어 **코드의 독립성(모듈화)을 떨어뜨립니다.**
    
*   따라서 디버그 관련 기능만 따로 모아둔 독립된 헤더 파일을 만들면, **딱 디버깅이 필요한 곳에서만 해당 헤더를 불러와 사용**할 수 있어 코드가 훨씬 깔끔하고 가벼워집니다.

## 2. 코드 리펙토링
### ① `DebugMacros.h` 작성
기존에 여러 군데 흩어져 있던 디버그 관련 코드와 매크로들을 새로 만든 헤더 파일 한곳으로 모읍니다. 안전한 컴파일을 위해 엔진의 디버그 헬퍼 헤더도 이쪽으로 옮겨옵니다.
```cpp
#pragma once

#include "DrawDebugHelpers.h" // 디버그 함수들이 정의된 엔진 헤더

// 1. 디버그 구체 매크로
#define DRAW_SPHERE(Location) if (GetWorld()) DrawDebugSphere(GetWorld(), Location, 25.f, 12, FColor::Red, true);

// 2. 디버그 선 매크로
#define DRAW_LINE(StartLocation, EndLocation) if (GetWorld()) DrawDebugLine(GetWorld(), StartLocation, EndLocation, FColor::Red, true, -1.f, 0, 1.f);

// 3. 디버그 포인트 매크로
#define DRAW_POINT(Location) if (GetWorld()) DrawDebugPoint(GetWorld(), Location, 15.f, FColor::Red, true);

// 4. 다중 행 벡터(선+점) 매크로
#define DRAW_VECTOR(StartLocation, EndLocation) \
if (GetWorld()) \
{ \
    DrawDebugLine(GetWorld(), StartLocation, EndLocation, FColor::Red, true, -1.f, 0, 1.f); \
    DrawDebugPoint(GetWorld(), EndLocation, 15.f, FColor::Red, true); \
}
```

### ② 소스 파일(`.cpp`)에서 사용하기
이제 전역 헤더(`Slash.h`) 대신, 우리가 만든 전용 헤더 파일만 인클루드해서 사용하면 됩니다.
```cpp
// Item.cpp 파일
#include "DebugMacros.h" // 전용 디버그 매크로 헤더만 깔끔하게 포함!

void AItem::BeginPlay()
{
    Super::BeginPlay();

    FVector Location = GetActorLocation();
    FVector Forward = GetActorForwardVector();
```
