# Trig Functions
## 1. 블랙박스로서의 사인(Sine) 함수
* 사인 함수는 하나의 '블랙박스(입력하면 출력이 나오는 상자)'로 생각할 수 있습니다.
* 어떤 값 _t_ 를 입력(_sin(t)_)하면, 사인 함수는 특정한 계산 과정을 거쳐 그에 대응하는 결과값을 우리에게 돌려줍니다.

## 2. 직각삼각형에서의 정의
직각삼각형에서 하나의 예각을 '세타(_thetaa_)'라고 지정했을 때, 사인과 코사인은 다음과 같은 변의 길이의 비율(Ratio)로 정의됩니다.

* **_sin(θ)_:** 대변(Opposite, 마주보는 변) / 빗변(Hypotenuse)
* **_cos(θ)_:** 인접변(Adjacent, 이웃한 변) / 빗변(Hypotenuse)

## 3. 단위원(Unit Circle)과 라디안(Radians)
### ① 단위원에서의 좌표 (X, Y)
반지름의 길이가 **1**인 원을 **단위원**이라고 합니다. 빗변의 길이가 1이 되므로 직각삼각형의 정의가 매우 단순해집니다.
* _sin(θ)_ = 대변(Opposite)/1 = Y 좌표
* _cos(θ)_ = 인접변(Adjacent)/1 = X 좌표
* 즉, 단위원 위를 움직이는 점의 _X_ 좌표는 코사인 값이 되고, _Y_ 좌표는 사인 값이 됩니다.

### ② 새로운 각도의 단위: 라디안 (Radians)
* 단위원의 둘레는 _2π_ 입니다 (둘레 = 2 x π x 반지름)
* 원을 한 바퀴 도는 것(360)은 단위원의 둘레인 **2π 라디안**과 같습니다.
    * 360 = 2π rad
    * 180 = π rad
    * 90 = π/2 rad
    * 45 = π/4 rad
        
* 컴퓨터 언어 및 게임 엔진의 삼각함수는 대부분 '도(Degrees)'가 아닌 '라디안(Radians)'을 입력값으로 받습니다.

## 4. 주기성과 사인/코사인 그래프 (Sinusoidal Motion)
위 그래프처럼 입력값 _t_ 가 증가함에 따라 두 함수는 특유의 부드러운 파동을 그리며 무한히 반복되는데, 이를 주기성(Periodicity)이라고 하며 이 패턴을 **사인파(Sinusoidal) 형태**라고 부릅니다.
* **사인 그래프 (_sin(t)_):** 0에서 시작하여 π/2(90도)일 때 최대값 1, π(180도)일 때 0, 3π/2(270도)일 때 최소값 -1을 찍고 2π(360도)에서 다시 0으로 돌아옵니다.
* **코사인 그래프 (_cos(t)_):** 1에서 시작하여 위와 같은 주기로 부드럽게 진동합니다.
* **두 그래프의 관계:** 사인과 코사인은 사실 완전히 같은 형태의 곡선입니다. 단지 코사인 그래프가 π/2(90도) 왼쪽으로 만큼 평행 이동(Shift)해 있을 뿐입니다.

## 5. 게임 개발에서의 활용 (결론)
* 게임이 실행되면 매 프레임마다 경과 시간(Delta Time)이 누적되어 시간 _t_ 변수가 계속 증가합니다.
* 이 시간 _t_ 를 사인 함수에 대입(_sin(t)_)하고, 그 결과값을 아이템이나 오브젝트의 **Z축(높이) 위치**에 더해주면 어떻게 될까요?
* 오브젝트가 위아래로 부드럽게 가라앉았다가 떠오르는 보빙(Bobbing) 액션(둥둥 떠 있는 효과)을 쉽게 구현할 수 있습니다.

# The Sine Function
## 1. 블루프린트에서 사인 함수 활용하기
매 프레임마다 시간의 흐름을 누적하는 '스톱워치' 역할을 할 변수를 만들어 사인 함수에 대입합니다.

* **시간 누적 변수 (_t_):** `Event Tick`이 실행될 때마다 기존 _t_ 값에 `Delta Seconds`(프레임 간 경과 시간)를 더해 다시 _t_ 에 저장합니다.
* **사인 노드 사용:** `Sign (Radians)` 노드를 배치하고 입력값으로 _t_ 를 연결합니다.
* **액터 위치 변경:** `Add Actor World Offset` 노드를 사용합니다. 입력 핀인 `Delta Location(Vector)`을 우클릭하여 `Split Struct Pin(구조체 핀 분할)`을 누르면 X, Y, Z 값을 따로 입력할 수 있습니다. 여기에 사인 함수 출력값을 **Z축**에 연결합니다.
* **수치 제어**
    * 사인의 결과값에 `곱하기(Multiply)`를 하면 위아래로 움직이는 높이(크기)를 조절할 수 있습니다.
    * 사인 함수 전 시간 _t_ 에 `곱하기`를 하면 위아래로 움직이는 속도를 조절할 수 있습니다.

## C++에서 `FMath::Sin` 사용하기
언리얼 엔진 C++에서는 **`FMath`라는 강력한 수학 라이브러리 클래스**를 제공하며, 여기에 포함된 정적(Static) 함수인 `Sin()`을 사용합니다.

### ① 헤더 파일 (`Item.h`) 변수 선언
클래스의 `private` 영역에 시간 누적 변수를 선언합니다.
```cpp
private:
    float RunningTime = 0.f; // 매 프레임 시간이 누적될 변수
```

### ② 소스 파일 (`Item.cpp`) 구현
`Tick` 함수 내부에서 기존의 X축 이동 및 회전 코드를 제거하고, 아래와 같이 상하 진동 코드를 작성합니다.
```cpp
void AItem::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // 1. 매 프레임 경과 시간을 RunningTime에 누적 (+= 연산자 사용)
    RunningTime += DeltaTime;

    // 2. FMath::Sin을 활용해 Z축 이동량 계산 (속도 5배, 크기 0.25배로 조절)
    float DeltaZ = FMath::Sin(RunningTime * 5.f) * 0.25f;

    // 3. 액터에 위치 오프셋 적용 (X:0, Y:0, Z:DeltaZ)
    AddActorWorldOffset(FVector(0.f, 0.f, DeltaZ));
}
```

## 3. 하드코딩 탈피: 진폭과 주기(시간 상수) 변수화
코드 안에 `0.25f`나 `5.f` 같은 숫자를 직접 써넣는 것을 하드코딩(Hard-coding)이라고 합니다. 이는 나중에 수치를 수정하기 어렵게 만들므로, 멤버 변수로 분리하는 것이 좋습니다.

* **진폭 (Amplitude):** 사인파가 위아래로 움직이는 최대 거리입니다.
* **시간 상수 (Time Constant / _k_):** 삼각함수에서 주기(Period)는 _2π/k_ 로 정의됩니다. 즉, 이 값이 커질수록 파동이 빨라집니다.

### ① 헤더 파일 (`Item.h`)에 변수 추가 및 초기화
```cpp
private:
    float RunningTime = 0.f;

    float Amplitude = 0.25f;      // 진폭
    float TimeConstant = 5.f;     // 시간 상수 (속도 조절)
```

### ② 소스 파일 (`Item.cpp`) 코드 수정
```cpp
// 수치 대신 변수명을 대입하여 유지보수를 쉽게 만듭니다.
float DeltaZ = FMath::Sin(RunningTime * TimeConstant) * Amplitude;
AddActorWorldOffset(FVector(0.f, 0.f, DeltaZ));
```


## C++ 팁: 멤버 변수를 초기화하는 3가지 방법과 효율성
강의에서는 C++에서 변수를 초기화하는 3가지 방법과 그에 따른 미세한 효율성 차이를 설명합니다.

1. **헤더 파일에서 선언과 동시에 초기화 (Variable Initialization)**
    * `float Amplitude = 0.25f;`
    * 변수가 생성되자마자 값이 할당되므로 가장 효율적입니다. (강사가 본 강의에서 채택한 방식)
        
2. **생성자 초기화 리스트 (Initializer List)**
    * `AItem::AItem() : Amplitude(0.25f) { ... }`
    * 1번과 마찬가지로 변수 생성과 동시에 초기화되므로 최고 효율을 냅니다.
        
3. **생성자 본문(Body)에서 대입 (Assignment)**
    * `AItem::AItem() { Amplitude = 0.25f; }`
    * 이는 '초기화'가 아니라 이미 쓰레기 값으로 생성된 변수에 값을 새로 '대입'하는 구조입니다. 즉, 단계가 하나 더 추가되므로 1, 2번에 비해 **미세하게 효율성이 떨어집니다.** (물론 현대 컴퓨터에서는 무시할 수 있는 수준의 차이입니다.)

# Exposing Variables to Blueprint
## 핵심 요약

C++에서 정의한 변수를 언리얼 에디터의 **상세(Details) 패널**에 나타나게 하려면 변수 위에 `UPROPERTY()` 매크로를 붙여야 합니다. 이 매크로 안에 어떤 **지정자**를 넣느냐에 따라 에디터에서 편집할 수 있는 범위가 달라집니다.

여기서 가장 중요한 개념은 블루프린트 원본(Default)과 레벨에 배치된 복사본(Instance)의 차이를 이해하는 것입니다.

### UPROPERTY 핵심 지정자 3가지
|지정자 (Specifier)|블루프린트 원본 (Default)|월드에 배치된 개체 (Instance)|특징 및 사용처|
|--|--|--|--|
|EditDefaultsOnly|정상 수정 가능|수정 불가능(숨겨짐)|원본 데이터만 일괄 수정하고 싶을 때 사용|
|EditInstanceOnly|수정 불가능 (숨겨짐)|정상 수정 가능|월드에 배치된 개체마다 각각 다른 값을 주고 싶을 때 사용|
|EditAnywhere|정상 수정 가능|정상 수정 가능|가장 흔히 사용. 어디서나 자유롭게 수정하고 싶을 때 사용|

## 원본(Default)과 인스턴스(Instance)의 작동 원리

강의 후반부에서 설명하는 **값의 상속과 덮어쓰기(Override)** 규칙은 다음과 같습니다.
* **상속 관계:** 블루프린트 원본(Default)에서 값을 바꾸면, 월드에 배치된 인스턴스들의 값도 자동으로 함께 바뀝니다.
    
* **독립적인 수정 (오버라이드):** 특정 인스턴스의 값을 에디터에서 직접 수정하는 순간, 그 인스턴스는 원본과의 연결이 끊어집니다. 이후 원본 값을 아무리 바꿔도 **해당 인스턴스의 값은 유지**됩니다.
    
* **되돌리기:** 인스턴스 창에서 변수 옆에 있는 **둥근 화살표(Reset to Default) 아이콘**을 클릭하면, 언제든지 다시 원본(Default)의 값으로 되돌릴 수 있습니다.

## 결론 및 장점
* **코드 컴파일 최소화:** C++ 코드를 매번 수정하고 컴파일할 필요 없이, 에디터에서 값을 마우스나 숫자로 즉시 바꾸며 테스트(기획 및 밸런스 조정)할 수 있습니다.
    
* **유연한 배치:** 같은 C++ 클래스로 만든 아이템이더라도, 어떤 것은 높게 움직이고(`Amplitude`), 어떤 것은 느리게 움직이도록(`Time Constant`) 개별적인 설정이 가능해집니다.

# Visible But Not Editable
## 값을 수정하면 안 될 때: `Visible` 지정자

`runningTime` 같은 변수는 C++ 코드에서 매 프레임 `DeltaTime`을 더하며 실시간으로 계산합니다. 이런 변수를 에디터에서 손으로 직접 수정해 버리면 게임 로직이 꼬일 수 있습니다.

이럴 때 사용하는 것이 **"기획자가 볼 수는 있되, 수정은 못 하게 회색(Read-Only)으로 잠그는"** `Visible` 지정자입니다. 특히 게임을 플레이(`Play`)하는 도중에 월드에 배치된 개체를 클릭하면, **값이 실시간으로 올라가는 것을 눈으로 디버깅**할 수 있어 매우 유용합니다.

## UPROPERTY 핵심 지정자 6종 최종 요약

언리얼 엔진의 데이터 제어는 크게 두 가지 질문으로 결정됩니다.
1. **편집도 시켜줄 것인가(`Edit`), 보기만 시켜줄 것인가(`Visible`)?**
2. **어디서 보여줄 것인가(`DefaultsOnly` / `InstanceOnly` / `Anywhere`)?**

|지정자 (Specifier)|편집 가능 여부|블루프린트 원본 (Default)|월드 배치 개체 (Instance)|실제 활용 예시|
|--|--|--|--|--|
|EditDefaultsOnly|가능 (Edit)|보임 + 편집|안 보임|모든 아이템에 일괄 적용할 기본 데미지 설정|
|EditInstanceOnly|가능 (Edit)|안 보임|보임 + 편집|특정 방에 있는 보물상자만 열리도록 하는 열쇠 ID|
|EditAnywhere|가능 (Edit)|보임 + 편집|보임 + 편집|(가장 흔함) 체력, 이동 속도 등 일반적인 스탯|
|VisibleDefaultsOnly|불가 (Read-Only)|보임 (수정 불가)|안 보임|원본 데이터의 고유 식별 번호(ID) 확인용|
|VisibleInstanceOnly|불가 (Read-Only)|안 보임|보임 (수정 불가)|해당 개체의 현재 좌표, 실시간 상태값 확인용|
|VisibleAnywhere|불가 (Read-Only)|보임 (수정 불가)|보임 (수정 불가)|(흔함) 실시간 흐른 시간(runningTime), 컴포넌트 참조|

## 꿀팁 및 핵심 포인트

* **실시간 디버깅의 강력함:** `VisibleInstanceOnly`나 `VisibleAnywhere`를 사용하면, 게임 플레이 중에 월드 아웃라이너(World Outliner)에서 배치된 인스턴스를 선택해 값이 실시간으로 변하는 모습(예: 35초, 36초...)을 상세 패널에서 바로 관찰할 수 있습니다.
    
* **C++ 개발자의 권한 제어:** 이 지정자들을 적절히 활용하면, 블루프린트를 사용하는 디자이너나 기획자가 실수로 중요 시스템 변수를 건드려 버그를 만드는 것을 원천 차단할 수 있습니다. 데이터의 **안전성**과 **유연성**을 동시에 챙기는 핵심 도구입니다.

# Exposing Variables to the Event Graph
## 이벤트 그래프 노출을 위한 UPROPERTY 지정자
이전의 `EditAnywhere` 등은 상세 패널 전용이었습니다. 이벤트 그래프에서 변수 노드를 사용하려면 `BlueprintReadOnly`나 `BlueprintReadWrite`를 추가해야 합니다.

|지정자 (Specifier)|이벤트 그래프 기능|특징 및 제약 조건|
|--|--|--|
|BlueprintReadOnly|Getter (Get) 노드만 생성|값을 읽을 수만 있고, 블루프린트에서 수정은 불가능함.|
|BlueprintReadWrite|Getter + Setter (Set) 둘 다 생성|블루프린트 그래프 안에서 값을 자유롭게 가져오고 수정함.|

> **중요 제약:** 기본적으로 이 지정자들은 C++의 `private` 섹션에 있는 변수에는 사용할 수 없으며, `protected`나 `public` 영역으로 변수를 옮겨야 컴파일 오류가 나지 않습니다. (`private`에서 쓰는 방법은 3번 참고)

## 변수 깔끔하게 정리하기: `Category`
변수가 많아지면 상세 패널과 블루프린트 변수 창이 지저분해집니다. 이럴 때 `Category` 지정자를 사용하면 에디터에 그룹을 만들어 예쁘게 분류할 수 있습니다.

```cpp
// 쉼표(,)로 구분하여 지정자를 이어 붙입니다.
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "SineParameters")
float Amplitude;
```
* **결과:** 에디터 상세 패널과 블루프린트 우측 변수 리스트에 "SineParameters"라는 그룹이 생기고 그 안에 변수가 정렬됩니다. 검색할 때도 카테고리 이름으로 모아볼 수 있어 관리가 매우 편해집니다.

## C++의 비밀(Private) 유지하기: `meta = (AllowPrivateAccess = "true")`
객체지향 프로그래밍 관점에서 변수를 외부에 함부로 노출하지 않으려고 `private`으로 유지하고 싶을 때가 있습니다. 하지만 블루프린트 그래프에서는 이 값을 가져와서 쓰고 싶다면 어떻게 해야 할까요?

언리얼 엔진은 이를 위해 아주 특수하고 강력한 **메타 데이터(Meta Specifier)** 문법을 제공합니다.

```cpp
private:
    // private이지만 블루프린트 이벤트 그래프에서 Get 노드로 쓸 수 있게 허용!
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, meta = (AllowPrivateAccess = "true"), Category = "SineParameters")
    float RunningTime;
```

* **`meta = (AllowPrivateAccess = "true")`**: 이 한 줄을 넣어주면, C++에서는 객체지향의 `private` 캡슐화를 철저히 지키면서도, 블루프린트 에디터에게만 "이 private 변수 접근을 특별히 허용해 줄게"라고 예외를 인정해 줍니다.
    
* 강의 속 `RunningTime`처럼 C++ 내부에서만 계산하고(Private), 블루프린트에서는 읽기만 해야 하는(ReadOnly) 핵심 데이터를 보호할 때 가장 완벽한 보안책이 됩니다.

## 요약 및 활용 예시 (감쇠 진동 구현)

강의에서는 `Amplitude`(진폭)를 `BlueprintReadWrite`로 노출한 뒤, 블루프린트 이벤트 틱(Event Tick)에서 아래와 같이 계산하는 예시를 보여줍니다.

1. 매 프레임 `DeltaTime`에 소량의 값을 곱해 진폭에서 뺍니다.
    
2. 값이 마이너스로 떨어지지 않도록 **`Clamp Float`** 노드를 사용해 최소 `0.0`, 최대 `1.0` 사이로 고정합니다.
    
3. 이 값을 다시 `Set Amplitude`로 저장하여 시간이 흐를수록 물체의 움직임이 줄어드는 감쇠 진동(Damped Oscillation)을 구현합니다.

# Exposing Funcitons to Blueprint
## C++ 함수를 블루프린트에 노출하는 2가지 방법

함수 위에 `UFUNCTION()` 매크로를 붙이고 내부에 지정자를 적어줍니다. 어떤 지정자를 쓰느냐에 따라 블루프린트 노드의 형태가 완전히 달라집니다.

### ① `BlueprintCallable` (실행 핀이 있는 노드)
* **특징:** 노드 좌우에 하얀색 실행 삼각형 핀(Execution Pin)이 생깁니다. 이벤트 흐름(틱, 시작 등)에 맞춰 순서대로 실행되어야 하는 함수에 사용합니다.
* **노드 색상:** 보통 **파란색** 노드로 나타납니다.
* **사용처:** 데이터를 변경하거나, 물리적인 액션을 취하는 등 게임 상태에 영향을 주는 함수.

### ② `BlueprintPure` (실행 핀이 없는 노드)
* **특징:** 실행 핀이 없고, 오직 입력 값과 출력(Return) 값만 존재합니다. 호출되는 순간 상태를 바꾸지 않고 **단순히 계산된 값만 반환**하는 함수에 사용합니다. (C++의 `const` 함수나 수학 계산 함수에 적합)
* **노드 색상:** 보통 **녹색** 노드로 나타납니다.
* **사용처:** `GetActorLocation`처럼 값을 가져오기만 하거나, 수학 공식 계산(예: 사인, 코사인 변환).

## 함수 접근 제한자(Access Specifier) 결정 팁

C++에서 함수를 선언할 때 `public`, `protected`, `private` 중 어디에 둘지 고민된다면 다음 기준으로 선택합니다.

* **`public`:** 외부의 다른 클래스/오브젝트가 이 함수를 호출해야 할 때.
* **`protected`:** 외부에선 못 부르게 막되, 이 클래스를 상속받을 자식 클래스(예: Item을 상속받은 Weapon)에서는 호출할 수 있게 하고 싶을 때. (**강의 추천: 잘 모를 때는 우선 protected에 두기**)
* **`private`:** 오직 이 클래스 내부에서만 사용할 때.

> ⚠️ **주의:** 변수와 마찬가지로 `private` 함수를 블루프린트에 노출하려면 컴파일 에러를 막기 위해 추가적인 설정이나 `protected`로의 이동이 필요할 수 있습니다.

## C++의 예약어 const가 BlueprintPure로 자동 처리되는 원리

C++에서 함수 멤버 뒤에 붙는 `const`는 "이 함수는 클래스의 멤버 변수(상태)를 절대 변경하지 않는 안전한 읽기 전용 함수다"라는 것을 컴파일러에게 보장하는 키워드입니다.

언리얼 엔진은 이 의미를 블루프린트 시스템에 그대로 대입합니다.
* **C++:** `float GetTransformedSign() const;`
* **언리얼 엔진의 해석:** "멤버 변수를 안 바꾸는 함수라고? 그럼 블루프린트에서도 실행 흐름 선(실행 핀)이 필요 없는 **Pure 노드**로 만들어도 되겠네!"

따라서 아래의 두 코드는 블루프린트 에디터에서 **완전히 동일한 녹색 Pure 노드**로 나타납니다.
```cpp
// 1번 방법: BlueprintPure를 명시함
UFUNCTION(BlueprintCallable, BlueprintPure)
float GetTransformedSign();

// 2번 방법: const를 사용함 (언리얼이 자동으로 Pure로 인식)
UFUNCTION(BlueprintCallable)
float GetTransformedSign() const;
```

## 단, 이것은 주의하세요! (암시적 변환의 예외)

`const`만 붙여도 Pure 노드가 되니까 코드가 더 깔끔해 보이지만, 결정적인 차이점이 딱 하나 있습니다. 바로 **함수의 반환값(Return Value) 유무**입니다.

### 1. 반환값이 없는 const 함수는?

만약 함수가 `void` 리턴형이거나 반환값이 없는데 `const`만 붙어있다면, 언리얼 엔진은 이를 Pure 노드로 만들지 않고 일반 Callable 노드(파란색 노드)로 만듭니다. 값을 돌려주지도 않는데 실행 핀까지 없으면 블루프린트에서 아예 호출할 방법이 없어지기 때문입니다.

### 2. 무조건 Pure 노드로 강제하고 싶다면?

가끔 반환값이 없더라도(예: `void`형 함수이지만 내부에서 Out 파라미터 `레퍼런스(&)`나 `포인터(*)`를 통해 값을 채워 반환하는 경우), 실행 핀이 없는 Pure 노드로 만들고 싶을 때가 있습니다. 이럴 때는 `const`만으로는 불가능하며, 반드시 **`BlueprintPure` 지정자를 명시**해 주어야 합니다.

## 요약 및 권장 스타일

* **자동 변환:** `BlueprintCallable`과 C++ `const`가 만나고 **반환값**이 존재하면, 언리얼이 자동으로 `BlueprintPure`처럼 취급합니다.
    
* **객체지향 관점에서의 이점:** C++ 코드 자체의 데이터 안정성(`const` 캡슐화)을 지키면서 블루프린트 노드까지 깔끔하게 녹색으로 뽑아낼 수 있으므로, 값을 단순 계산하거나 가져오는 Getter 함수에는 **`const`를 적극적으로 활용하는 것이 아주 좋은 습관**입니다.
