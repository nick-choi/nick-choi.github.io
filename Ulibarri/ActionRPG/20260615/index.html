SetActorLocation¶

1. 블루프린트에서 액터 위치 설정하기¶

  • 사용 노드: Begin Play 노드에서 선을 드래그해 Set Actor Location 함수를 검색하여 배치합니다.

  • 주요 입력 값

    • Target (타겟): 기본값은 Self로 설정되어 있어, 이 코드가 실행되는 액터 자신을 가리킵니다.
    • New Location (새 위치): X, Y, Z 축 값을 직접 입력하거나 벡터 값을 연결할 수 있습니다. 강의에서는 X = 0, Y = 0, Z = 200 으로 설정했습니다.

  • 알 수 있는 사실 (실행 순서)

    • 게임을 실행하면 액터들이 모두 지정된 위치(0, 0, 200)로 이동합니다.
    • 이를 통해 블루프린트의 Begin Play가 C++의 BeginPlay보다 먼저 실행된다는 중요한 사실을 알 수 있습니다. (블루프린트에서 먼저 위치를 옮긴 후, C++에서 디버그 구체와 선을 그렸기 때문입니다.)

2. ## C++에서 액터 위치 설정하기 (SetActorLocation)¶

블루프린트 노드를 지우고, 동일한 기능을 C++ 코드로 구현하는 과정입니다.

  • 함수 호출: 디버그 구체를 그리기 직전에 SetActorLocation() 함수를 호출합니다.
  • 위치 값 전달: 기존 위치를 그대로 넣으면 변화가 없으므로, 새 위치를 지정해야 합니다.
    • 변수를 따로 만들지 않고, FVector 생성자를 사용해 임시 벡터 객체를 인자로 바로 전달합니다.
      SetActorLocation(FVector(0.0f, 0.0f, 50.0f));

매개변수(Input)에 대한 팁¶

SetActorLocation 함수는 위치 값 외에도 여러 매개변수(bSweep, FHitResult, ETeleportType)를 가지고 있습니다. 하지만 C++에서는 기본값(Default Values)이 설정되어 있으므로, 위치 값(FVector)만 넣어주어도 정상적으로 작동합니다.

  • bSweep (스위핑): 이동 경로에 다른 물체가 있는지 감지하는 기능입니다. 켜져 있으면 장애물 바로 앞에서 이동을 멈춥니다. 강의에서는 기본값(거짓)을 그대로 사용합니다.

3. 디버그 도형 위치 수정 및 테스트¶

C++은 코드의 실행 순서(위에서 아래로)가 엄격하므로 주의해야 합니다.

  1. 순서 변경의 필요성: 기존에는 액터 위치를 옮기기 전에 location 변수(옛날 위치)를 받아와 디버그 구체를 그렸습니다. 만약 순서를 바꾸지 않으면, 액터는 새 위치로 가고 디버그 도형은 옛날 위치에 그려지게 됩니다.

  2. 해결 방법: SetActorLocation() 코드를 실행한 뒤, 그 아래에서 새 위치를 기준으로 디버그 구체(DrawSphere)와 선을 그리도록 코드 순서를 정렬합니다.

  3. 결과 확인: 게임을 플레이하면 액터가 새 위치(0, 0, 50)로 잘 이동하고 디버그 구체도 그 위치에 정확히 그려지는 것을 확인할 수 있습니다.

SetActorRotation¶

1. 회전의 3가지 축 (Pitch, Yaw, Roll)¶

언리얼 엔진에서 회전 값은 Rotator(로테이터) 구조체를 사용하며, X, Y, Z 축을 기준으로 각각 Roll, Pitch, Yaw라고 부릅니다.

  • Roll (X 축 기준 회전)
    • 비행기가 좌우로 기우는 듯한 회전 (앞뒤를 축으로 회전)
  • Pitch (Y 축 기준 회전)
    • 비행기 앞코가 위/아래로 들리는 듯한 회전 (좌우를 축으로 회전)
  • Yaw (Z 축 기준 회전)
    • 자동차가 좌/우회전하듯 수평으로 도는 회전 (상하를 축으로 회전)

강의에서는 액터를 수평으로 돌리기 위해 Yaw(Z축) 값을 변경합니다. 예를 들어 Yaw를 90으로 설정하면 시계 방향으로 90도 회전하고, -90으로 설정하면 반대 방향으로 회전합니다.

2. 블루프린트에서 액터 회전 설정하기¶

  • 사용 노드: Begin Play 노드에서 드래그하여 Set Actor Rotation 함수를 배치합니다.

  • Teleport Physics (물리 텔레포트) 옵션:

    • 기본값은 체크 해제(False) 상태입니다.
    • 체크 해제 (False): 액터가 갑자기 회전할 때, 그 움직임(속도)이 액터에 매달린 물리 컴포넌트(예: 캐릭터의 머리카락, 옷, 사슬, 로프, 래그돌 등)에 그대로 전달됩니다. 따라서 회전할 때 이 매달린 부품들이 출렁거리거나 펄럭이게 됩니다.
    • 체크 설정 (True): 물리 상태를 무시하고 액터의 회전값만 '순간이동' 시킵니다. 속도가 전달되지 않으므로 매달려 있는 머리카락이나 사슬이 출렁거리지 않고 얌전하게 위치만 이동합니다.

3. C++에서 액터 회전 설정하기 (SetActorRotation)¶

블루프린트 노드를 지우고 C++ 코드로 구현하는 과정입니다.

  • 함수 호출: SetActorRotation() 함수를 호출합니다. C++에서는 두 가지 버전(오버로딩)이 존재하는데, 하나는 쿼터니언(FQuat)을 사용하는 것이고, 다른 하나는 우리가 사용할 FRotator를 사용하는 것입니다.

  • FRotator의 생성자 주의점:

    • C++에서 FRotator 생성자에 인자를 때, 입력 순서가 블루프린트 창에 보이는 순서(X, Y, Z)와 다릅니다!

    • C++ FRotator 생성자는 (Pitch, Yaw, Roll) 순서로 값을 받습니다. (매우 중요)

      // 순서: Pitch(Y), Yaw(Z), Roll(X)
// 아래 코드는 Pitch = 0, Yaw = 45, Roll = 0 으로 설정하여 Z축 기준으로 45도 회전시킵니다.
SetActorRotation(FRotator(0.0f, 45.0f, 0.0f));

C++에서의 Teleport 옵션¶

블루프린트에서는 Teleport Physics가 단순한 Bool(체크박스)이었지만, C++에서는 ETeleportType이라는 열거형(Enum) 타입을 두 번째 인자로 받습니다. 스크립트에서는 생략했기 때문에 기본값(텔레포트 하지 않음)이 적용됩니다.

Actor World Offset¶

1. 월드 오프셋(World Offset)이란?¶

기존의 SetActorLocation이 특정 절대 좌표로 액터를 '순간이동' 시켰다면, AddActorWorldOffset은 현재 위치를 기준으로 원하는 만큼 '더해주는(이동시키는)' 함수입니다.
* Delta Location (입력값): 변경할 위치의 변화량(벡터)을 의미합니다. 예를 들어 Z 축에 200을 넣으면 현재 위치가 어디든 상관없이 위로 200 만큼 더 이동합니다.

  • AddActorWorldRotation: 회전도 마찬가지로 기존 회전값에 원하는 각도(오프셋)만큼을 더해줍니다. (예: Pitch에 45를 더하면 45도만큼 위를 올려다보게 됨)

2. 매 프레임 관찰을 위한 '싱글 프레임 디버그 매크로' 작성¶

기존 디버그 매크로는 선이 남아있는 Persistent Lines 방식이어서, 매 프레임 움직이는 액터에 사용하면 화면이 온통 빨간 구체로 도배됩니다. 따라서 그 프레임에만 잠깐 켜졌다 사라지는 싱글 프레임용 매크로를 DebugMacros.h에 새로 정의합니다.

  • 핵심 변경점

    • 지속성 옵션(bPersistentLines)을 true에서 false로 변경합니다.
    • 수명(Lifetime) 인자에 -1.0f를 전달하면 정확히 '1프레임' 동안만 유지됩니다.

  • 생성한 매크로

    • DrawSphereSingleFrame, DrawLineSingleFrame, DrawPointSingleFrame, DrawVectorSingleFrame

3. Tick 함수와 프레임 타임의 문제점¶

C++의 Tick 함수 내부에서 매 프레임 X축으로 1.0f씩 이동하게 코드를 짜고 테스트를 해봅니다.

// 매 프레임 X축으로 1cm씩 이동 (문제 발생 코드!)
AddActorWorldOffset(FVector(1.0f, 0.0f, 0.0f));

* 문제점: 컴퓨터 사양에 따라 프레임레이트(FPS)는 달라집니다.
* 사양이 좋아 120 FPS가 나오는 컴퓨터는 1초에 Tick이 120번 실행되므로 1초에 120cm를 이동합니다.
* 사양이 안 좋아 30 FPS로 고정된 컴퓨터는 1초에 Tick이 30번만 실행되므로 1초에 30cm만 이동합니다.
* 결론: 게임 사양(FPS)에 따라 캐릭터의 이동 속도가 달라지는 심각한 버그가 발생합니다.

4. 델타 타임(Delta Time)을 이용한 프레임 독립성 해결¶

이 문제를 해결하기 위해 Tick 함수의 인자인 DeltaTime(이전 프레임부터 현재 프레임까지 걸린 시간, 초 단위)을 활용해야 합니다. 이동 속도나 회전 속도에 DeltaTime을 곱해주면 어떤 프레임 환경에서든 언제나 일정한 속도로 움직이게 됩니다.

수학적 단위 계산 원리¶

  • 이동 속도(Movement Rate): 50cm/s (초당 50cm 이동목표)

  • 델타 타임(Delta Time): s/frame (프레임당 걸린 초)

  • 공식: Movement Rate x Delta Time = cm/s * s/frame = cm/frame

    • 초당위가 서로 약분되어 사라지고, '이번 프레임에 가야할 정확한 센티미터'만 남게 됨

C++ 구현 코드¶

속도와 회전에 모두 DeltaTime 을 곱하여 프레임 독립적인 움직임을 완성합니다.

void AItem::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // 1. 프레임 독립적 이동 (초당 50cm 속도)
    float MovementRate = 50.0f;
    AddActorWorldOffset(FVector(MovementRate * DeltaTime, 0.0f, 0.0f));

    // 2. 프레임 독립적 회전 (초당 45도 Yaw 회전)
    float RotationRate = 45.0f;
    AddActorWorldRotation(FRotator(0.0f, RotationRate * DeltaTime, 0.0f));

    // 3. 싱글 프레임 디버그 구체 그리기
    DrawSphereSingleFrame(GetActorLocation());

    // 4. 싱글 프레임 디버그 화살표(앞방향 벡터) 그리기 (길이 100 뻥튀기)
    DrawVectorSingleFrame(GetActorLocation(), GetActorLocation() + (GetActorForwardVector() * 100.0f));
}
