2. AddMovementInput¶

  //3인칭
  //Left/Right
  FRotator controlRotation = GetControlRotation();
  controlRotation.Pitch = 0;
  AddMovementInput(FRotationMatrix(controlRotation).
                   GetUnitAxis(EAxis::Y), xAxisValue);

  //Forward/Backward
  controlRotation = GetControlRotation();
  controlRotation.Pitch = 0;
  controlRotation.Roll = 0;
  AddMovementInput(FRotationMatrix(controlRotation).
                   GetUnitAxis(EAxis::X), yAxisValue);

4. SetupPlayerInputComponent¶

    if (auto enhancedInputComponent = Cast<UEnhancedInputComponent>(PlayerInputComponent))
    {
        if (MoveAction)
        {
            enhancedInputComponent->BindAction(MoveAction, ETriggerEvent::Triggered, this, &AShooterCharacter::Move);
        }

        if (JumpAction)
        {
            enhancedInputComponent->BindAction(JumpAction, ETriggerEvent::Started, this, &ACharacter::Jump);
            enhancedInputComponent->BindAction(JumpAction, ETriggerEvent::Completed, this, &ACharacter::StopJumping);
        }
    }

6. 그외¶

  • inputActionValue.Get<FVector2D>()
    • 입력장치의 조작량
    • X : 가로(좌우 이동량)
    • Y : 세로(위아래 이동량)
  • EAxis
    • 언리얼 엔진 3D 월드 방향
    • X : 앞/뒤(Forward/Backward)
    • Y : 좌/우(Left/Right)
    • Z : 위/아래(Up/Down)

157 Third Person Camera Spring Arm¶

C++에서 구현했을 때의 장점¶

  • 일반적인 현업(AAA 게임 개발이나 규모가 있는 프로젝트)에서는 압도적으로 C++에서 SpringArm과 CameraComponent를 생성하여 붙이고(계층 구조 설계), 블루프린트에서는 세부 수치만 조절하는 방식을 선호

1. C++ 로직에서의 안전한 접근성 (가장 큰 이유)¶

  • 슈팅 게임(ShooterCharacter)의 경우 카메라의 위치나 바라보는 방향(Rotation)을 기반으로 총알이 날아가는 궤적(Line Trace)을 계산하거나, 조준(Aim) 시 카메라의 시야각(FOV)이나 SpringArm의 길이를 조절하는 등의 로직이 C++에 구현되는 경우가 많습니다.

2. 블루프린트 손상(Corruption) 방지¶

  • 언리얼 엔진을 사용하다 보면 드물게 블루프린트 에셋이 꼬이거나 손상되는 경우가 발생합니다. 핵심 컴포넌트 구조(캡슐, 메시, 스프링암, 카메라 등)를 C++에 하드코딩해 두면, 블루프린트가 깨지더라도 부모 C++ 클래스를 기반으로 블루프린트를 다시 만들고 수치만 다시 입력하면 되기 때문에 복구가 훨씬 수월합니다.

3. 명확한 설계 의도 파악¶

  • 프로그래머가 C++ 헤더 파일만 열어봐도 "아, 이 캐릭터는 스프링암 기반의 3인칭 카메라를 사용하는 캐릭터구나"라는 것을 한눈에 파악할 수 있어 협업과 유지보수에 매우 유리합니다.

1. 컴포넌트 선언부¶

    //SpringArm
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Camera")
    class USpringArmComponent* SpringArm;

    //FollowCamera
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Camera")
    class UCameraComponent* FollowCamera;

2. 생성자에서 컴포넌트 생성/부착¶

    //SpringArm 생성
    SpringArm = CreateDefaultSubobject<USpringArmComponent>(TEXT("CameraBoom"));
    SpringArm->SetupAttachment(RootComponent);
    SpringArm->TargetArmLength = 300.0f;
    SpringArm->bUsePawnControlRotation = true; //카메라가 캐릭터를 따라 회전하도록 설정

    //CameraComponent 생성
    FollowCamera = CreateDefaultSubobject<UCameraComponent>(TEXT("FollowCamera"));
    FollowCamera->SetupAttachment(SpringArm, USpringArmComponent::SocketName); //SpringArm에 카메라 부착

    //SpringArm에서 컨트롤   회전을 사용하지만, 카메라에서는 사용하지 않도록 설정
    FollowCamera->bUsePawnControlRotation = false;
